## 🟡 中優先度（UXの改善）

### 1. フォームバリデーションが不十分
- 現状：必須チェックのみ
- 修正案：`react-hook-form` + `zod` の導入
  ```bash
  npm install react-hook-form zod @hookform/resolvers
  ```

### 2. スタッフ管理ページが未実装
- 現状：`staffs` テーブルはあるがUIがない
- シードデータのスタッフ（id=1, 2）が各所でハードコードされている
- 修正案：`/staff` ページを追加、スタッフ選択UIを各フォームに追加

### 3. ページネーションが未実装
- 現状：全件取得（利用者が増えると遅くなる）
- 修正案：APIに `page` / `limit` クエリパラメータを追加

### 4. CSP が緩い（unsafe-inline / unsafe-eval を許可）
- 現状：Next.js 本番ビルドとの互換性のため緩めの固定 CSP（`middleware.ts`）
- 修正案：Oracle 移行時に nonce ベースに戻す

## 🟢 低優先度（機能追加・改善）

### 5. Excel 転記の対応シート拡大
- 現状：別紙１・1_1計画案 のみ（`SUPPORTED_SHEETS`）
- 修正案：別紙２・2_1計画・3_1モニタ などへ拡大

### 6. PDFエクスポートが未実装
- 修正案：`@react-pdf/renderer` または `jspdf` でPDF出力
- 用途：支援計画書・モニタリング報告書の印刷

### 7. 支援計画書の詳細フォームが未実装
- 現状：AI生成テキストのみ表示
- 修正案：生成したテキストを編集・保存できるフォームを追加

### 8. 通知機能が未実装
- モニタリング期日が近くなったらメール通知
- 修正案：cronジョブ + smtplib（Python標準）

### 9. AIチャットの会話管理が簡素
- 現状：ページを離れると同じ session_id が使われ続ける
- 修正案：セッション一覧画面を作り、過去の会話を選択・削除できるようにする
- 関連：会話履歴のトリミング（古いメッセージを削除する仕組み）も未実装

### 10. 支援記録の添付ファイル機能が未実装
- 修正案：`python-multipart` はすでに入っているのでバックエンドは対応可能

### 11. 統計・レポート機能が簡素
- ダッシュボードの数字だけ
- 修正案：月別グラフなど（`recharts` ライブラリが shadcn に対応）
